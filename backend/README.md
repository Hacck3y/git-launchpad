# Git Launchpad Backend

FastAPI backend that handles deploying GitHub repos into Docker containers with AI-powered stack detection, companion service provisioning, and automated error recovery.

## Quick Start

```bash
# Install
pip install -r requirements.txt

# Run
python main.py
# or
uvicorn main:app --host 0.0.0.0 --port 8000
```

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/deploy` | Start a deployment |
| GET | `/api/deploy/{id}` | Get deployment status |
| DELETE | `/api/deploy/{id}` | Kill and clean up a deployment |
| WS | `/ws/logs/{id}` | Real-time log streaming (WebSocket) |
| GET | `/docs` | Swagger API documentation (auto-generated) |

## Deployment Lifecycle

```
cloning → building → running → health_check → live
                  ↓                              ↓
              ai_fixing (up to 3 retries)     expired / killed
                  ↓
               error (after max retries)
```

## Companion Services

The engine automatically detects and provisions companion containers:

| Service | Docker Image | Default Port | Auto-detected via |
|---------|-------------|-------------|-------------------|
| MySQL | `mysql:8.0` | 3306 | `DATABASE_URL`, `MYSQL_*`, `DB_*` env vars |
| PostgreSQL | `postgres:16-alpine` | 5432 | `DATABASE_URL`, `POSTGRES_*`, `PG_*` env vars |
| MongoDB | `mongo:7` | 27017 | `MONGO_*`, `MONGODB_*` env vars |
| Redis | `redis:7-alpine` | 6379 | `REDIS_*`, `CACHE_*` env vars |

Services are connected via an isolated Docker bridge network. The engine patches application config files to replace `localhost`/`127.0.0.1` references with the service hostname.

## Configuration Patching

The engine scans and patches these file types to fix hardcoded connection strings:

| File Types | Patterns Replaced |
|-----------|-------------------|
| `.js`, `.ts`, `.json`, `.py`, `.rb`, `.php` | `localhost:PORT` → `service:PORT` |
| `.yml`, `.yaml`, `.toml`, `.cfg`, `.ini` | `host: 'localhost'` → `host: 'service'` |
| `.env`, `.env.*` | `127.0.0.1` → service hostname |

> **Note:** Elixir files (`.ex`, `.exs`) are excluded from regex patching to prevent syntax corruption.

## AI Auto-Fix

When a build or runtime health check fails, the engine:

1. Collects error logs and repo context
2. Sends them to the AI fix endpoint (Gemini via edge function)
3. Applies the suggested Dockerfile and env changes
4. Retries the build (up to 3 attempts)

## Resource Limits

| Resource | Limit |
|----------|-------|
| Memory per container | 512 MB |
| CPU per container | 1.0 core |
| Preview port range | 10000–10100 |
| Max AI fix retries | 3 |

## Container Lifetime (TTL)

| Plan | Duration |
|------|----------|
| Free | 20 minutes |
| Premium | 60 minutes |
| Elite | Unlimited |

The `CleanupManager` runs in a background thread and removes expired containers automatically.

| `cleanup.py` | Background TTL expiry and container cleanup |
